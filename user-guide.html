1. What it does#

Veeam Advisor is a single-file, browser-based tool that takes a Veeam Backup & Replication (VBR) VMC.log file and turns it into two things:

  • Sizing recommendations — proxy, repository, backup-server and VSA sizing, with NIC-bandwidth modelling, GFS retention, and a storage-growth projection.
  • A best-practice / security compliance review — 50+ findings drawn from bp.veeam.com/security and bp.veeam.com/vbr, plus a full inventory of what’s in the environment.

It auto-detects whether the server is a VSA (Veeam Software Appliance) or a Windows VBR install, the virtualisation platform (VMware / Hyper-V / Proxmox / Nutanix AHV), Cloud Connect roles (provider or tenant), proxies, repositories, every job type, replication, agent jobs, failover plans, GFS/retention, malware events, MFA, config-backup encryption, Enterprise Manager, tape infrastructure, and NIC speed. If Veeam Backup for Azure is linked alongside VBR, it picks that up too.

You can also run it with no log at all — type your numbers into the form and use it purely as a sizing calculator. A log simply adds the auto-population and the compliance findings on top.

2. How it handles your data (privacy)#

Everything runs entirely in your browser. The VMC.log you load is never uploaded, transmitted, or stored anywhere — there is no server-side runtime. All values parsed from the log are HTML-escaped before they are displayed.

In practice this means you can run a customer’s log through it on your own machine without the file leaving your laptop. It also means there is nothing to save online: when you close the tab the parsed data is gone, so export a PDF if you want a record.

3. Getting your input data#

You have two ways to feed the tool. Either is fine.

Option A — a VMC.log file

VMC.log is one of the diagnostic / inventory logs produced by a VBR server. If you did not collect it yourself, ask whoever manages the VBR environment to provide the current one. On the VBR server, Veeam logs live under the Veeam program-data log folder (typically C:\ProgramData\Veeam\Backup\…); confirm the exact path for your version if you are collecting it yourself.

A single VMC.log can contain several appended collection runs. Veeam Advisor automatically reads the run with the latest date/time and shows that collection date in the UI and on the PDF, so you always know how current the data is.

Accepted formats: .log and .txt (including RTF exports), from VBR 12.x and 13.x, on Hyper-V or VMware.

Option B — the companion PowerShell script (live pull)

If you have access to the VBR server and would rather pull fresh data directly, the repo ships VeeamAdvisor-PowerShell.ps1. It collects the same inventory the app consumes (including a live coverage / orphaned-job analysis) straight from the Veeam API.

  • PowerShell 5.1+ for VBR v12  ·  PowerShell 7.4.7+ for VBR v13
  • Enforces TLS 1.2+; connects to a local console, a remote VBR server, or a VSA by hostname/IP (prompts for credentials when remote)
# Run locally on the VBR server
.\VeeamAdvisor-PowerShell.ps1

# Target a remote server with explicit credentials
.\VeeamAdvisor-PowerShell.ps1 -VBRServer vbr01.lab.local -Credential (Get-Credential)

4. Quick start#

  1. Open Veeam Advisor in a browser (Chrome recommended — see Exporting a PDF for why).
  2. Drag a VMC.log onto the Import VMC.log box, or click it to browse. The form fills in automatically, and the auto-detected fields turn green.
  3. Click Provide recommendations. The right-hand panel populates with sizing plus the tabbed analysis.
  4. (Optional) Click Export PDF in the top bar for a shareable report.

No log handy? Skip straight to typing values into Environment parameters and click Provide recommendations — you will get sizing without the compliance findings.

5. The interface, panel by panel#

Top bar

  • Version toggle (v12.x / v13.x): tells the tool which VBR generation you are sizing for. Defaults to v13.x. See v12.x vs v13.x.
  • Export PDF: disabled (greyed out) until you have run a calculation — that is expected, not a bug.

Left column

  • Import VMC.log — the drag-and-drop zone. After a successful parse it shows a “VMC.log parsed — fields populated” panel summarising what it found.
  • Environment parameters — the form that drives sizing, grouped into Infrastructure, Backup policy, GFS retention, and Sizing & deployment. A Reset button clears it.

Right column

  • Sizing recommendations — empty until you calculate; then it holds the headline sizing plus all 15 analysis tabs.

6. Step 1 — Import a VMC.log#

Drag the file onto the box or click to browse. On success:

  • Fields the tool could read from the log are filled in and highlighted green with an auto badge.
  • A summary panel confirms the parse and shows key detected facts.
  • The collection date/time of the latest run in the file is recorded (and later printed on the PDF).

If the file is malformed or unreadable, you get a clear on-screen message rather than a silent failure — re-export the log or check you grabbed the right file.

7. Step 2 — Review the environment parameters#

Green fields came from the log; you can override any of them. Anything not stored in a VMC.log is left for you to enter. The groups:

  • Infrastructure — Total VMs · Source data (TB) · Hosts · Clusters · Backup jobs · Existing proxies.
  • Backup policy (from VMC.log) — Retention (days) · Daily change rate % · Backup mode (forward / forever-forward incremental, active full + incrementals, synthetic full weekly, or reverse incremental).
  • GFS retention (from VMC.log) — Daily · Weekly · Monthly · Yearly restore points.
  • Sizing & deployment — Backup window (hours) and Proxy NIC speed.

NIC link speed is not stored in VMC.log, so the adapter is detected but its speed must be set manually (1 GbE up to 100 GbE, or “Not applicable” for HotAdd/SAN). Set this to match your actual proxy hardware — it directly affects the throughput model.

A couple of policy fields are version-aware. Reverse incremental is flagged as legacy when you are in v13 mode (see v12.x vs v13.x).

8. Step 3 — Generate recommendations#

Click Provide recommendations. The right panel fills with:

  • Headline sizing (proxies, repository capacity, backup-server resources, VSA where relevant).
  • The 15 analysis tabs (next section), each with counts/badges where the log surfaced something noteworthy.

Re-run it any time after changing a field — it recalculates from the current form values. Running a calculation is also what enables the Export PDF button.

9. Reading the results: the 15 tabs#

Tabs carry small badges/counts when there is something to flag (for example a replication count, a malware count, or the number of BP findings). Tabs in app order:

#TabWhat it shows
1VSASizing for the Veeam Software Appliance. v13-only — in v12 mode it tells you to use VBR Windows instead.
2VBR WindowsSizing for a Windows-based backup server.
3Proxy BPProxy sizing and best-practice guidance (count, transport mode, concurrency).
4RepositoryPer-repository detail: type, immutability (incl. Hardened Linux and Veeam Data Cloud), task limits, per-VM chains, dedup/compression flags.
5RetentionShort-term retention plus the GFS (daily/weekly/monthly/yearly) picture.
6ReplicationReplication jobs and failover plans.
7All jobsEvery parsed job across all types, including orphaned/disabled jobs that create no restore points.
8Cloud ConnectVCSP provider or tenant detail when the server has a Cloud Connect role.
9MalwareMalware-detection events broken down by type, source, state and severity, with file-extension detail.
10SettingsConfiguration findings (warnings) — MFA, config-backup encryption and similar global settings.
11BP ReviewThe consolidated best-practice / security review and overall BP score.
12AgentsVeeam Agent protection, distinguishing managed agent backups, agent policies, and legacy standalone jobs.
13TapeExisting tape infrastructure (servers, drives, libraries, media pools) and a single GFS tape-job sizing across LTO-7/8/9.
14InfrastructureOne-screen inventory mapping Veeam BP design areas to what was parsed, plus a security-posture summary (MFA / immutability / encryption).
15LicensingLicences consumed vs available with a headroom gauge, per-workload breakdown, capacity (VUL) usage, and renewal/expiry advisories.

On protection coverage: the tool compares discovered infrastructure VMs against VMs covered by jobs and Veeam Agent backups and flags an estimated unprotected count. Treat it as a prompt to investigate — for the exact list of unprotected VMs, use the Veeam ONE Protected VMs report.

10. Exporting a PDF report#

Once you have calculated, click Export PDF (top bar). The report includes both the date it was generated and the data-collection date from the log, so a reader can tell how current it is. The report flows continuously — headings stay with their tables and content blocks do not split awkwardly across pages.

Pick the right print destination. In the print dialog, choose “Save as PDF” (Chrome’s built-in option). This produces a vector, searchable, roughly 4× smaller file. Avoid “Microsoft Print to PDF”, which rasterises the page into a large, image-only PDF.

11. v12.x vs v13.x — the version toggle#

The toggle in the top bar tells Veeam Advisor which VBR generation you are sizing for, and it changes some guidance:

  • VSA (Veeam Software Appliance) requires v13. In v12 mode the VSA tab is unavailable and points you to VBR Windows.
  • Reverse incremental is deprecated for new jobs in v13. It stays selectable so you can size existing upgraded/legacy jobs, but new jobs should use forward incremental, active full, or synthetic full.
  • Running v12 surfaces a note that v13-only features — VSA, improved Hardened-Linux immutability, and several newer BPA/security checks — are not available, with a pointer toward planning an upgrade.

Set it to match the environment you are actually advising on before you calculate.

12. Fleet View (multiple servers)#

Veeam Advisor analyses one server in depth. For a whole estate, the repo includes a companion file, Veeam_Advisor_Fleet.html (Fleet View). You upload one VMC.log per server and click Build fleet view for:

  • Fleet Dashboard — estate totals, fleet-wide coverage, average BP score, and a per-server table sorted weakest-first.
  • Comparison Matrix — servers as columns and BP checks / key values as rows, with inconsistent rows highlighted so you can spot odd-one-out configs.
  • Aggregate Report — combined posture and a remediation-priority list, weakest servers first.

Note that Veeam Advisor and Fleet View are separate codebases that parse the same log format. The newest additions (for example Veeam Backup for Azure detection and a few licensing-consumption corrections) currently live in Veeam Advisor first. If a finding appears in one tool but not the other for the same log, that is usually why — check which tool you are in before assuming a discrepancy is a bug.

13. Troubleshooting & FAQ#

The Export PDF button is greyed out.
You have not calculated yet. Click Provide recommendations first.
My PDF is huge and the text is not selectable.
You printed via Microsoft Print to PDF. Re-export using Chrome’s Save as PDF.
Nothing populated after I loaded the file.
Make sure it is an actual VMC.log (.log / .txt / RTF) from VBR 12.x/13.x. A malformed file shows an error message — re-collect and try again, or use the PowerShell companion to pull data live.
NIC speed looks wrong / the throughput numbers seem off.
NIC link speed is not recorded in VMC.log. Set Proxy NIC speed manually to match your hardware, then recalculate.
It says VSA is not available.
You are in v12.x mode. Toggle to v13.x (VSA is v13-only), or size with the VBR Windows tab for your current version.
The unprotected-VM count surprises me.
It is an estimate based on parsed jobs and agents. Confirm against the Veeam ONE Protected VMs report before acting.
Can I use it without a log?
Yes. Type into Environment parameters and calculate for sizing only; the compliance findings need a log.
Is my customer’s data safe?
Yes — it is processed locally in your browser and never uploaded. Nothing persists after you close the tab, so export a PDF if you need a record.

14. Limitations#

Veeam Advisor is an unofficial aid for sizing and best-practice review, not an authoritative source. Its outputs are estimates from a parsed log; NIC speed and a few other values are not in the log and depend on what you enter. Always validate against the official Veeam calculators, the Veeam Help Center, and Best Practices documentation, and involve a Veeam SE / Solution Architect for final decisions.
